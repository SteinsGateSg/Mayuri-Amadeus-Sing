Mayuri-Amadeus-Sing

Some voices remain
only as light on water.

Mayuri-Amadeus-Sing is a character-focused singing voice conversion framework built on RVC + Applio, shaped around the idea that a remembered voice is more than timbre alone. It is timing, air, distance, and the emotion that remains after the note has faded.

This release preserves the workflow, the demo model, and one complete public example based on シリウスの心臓. The aim is not to freeze a single cover, but to offer a reproducible path for training, evaluating, and refining character singing systems.

Dataset SteinsGateSg/mayuri-voice-dataset
Demo Model mayuri_rvc_v1_200e_55200s.pth
Backend Applio 3.5.1 with the RVC family workflow
Public Example Full vocal and full mix demos from the Sirius no Shinzou case

A character voice is not remembered because it is loud. It is remembered because it carries a fragile world within it, and leaves that world behind when the song is over.

Start

  • bash scripts/bootstrap_env.sh
  • bash scripts/install_backend_deps.sh
  • python scripts/link_existing_dataset.py --raw-source /path/to/wav
  • bash scripts/run_applio.sh

The first UI launch or first CLI inference downloads Applio runtime prerequisites such as rmvpe.pt and contentvec.

Music/ is kept as a placeholder. Local experiments should place a guide vocal and instrumental under inputs/.

Included

  • Applio 3.5.1 backend source prepared for local setup
  • One kept Mayuri demo checkpoint and matching index
  • Scripts for dataset linking, UI launch, runtime preparation, and inference sweeps
  • Publishable demo audio for vocal-only and full-mix listening

Intentionally Omitted

  • Raw training WAV archives
  • Original song stems and local music assets
  • Training caches, transient checkpoints, and downloaded runtime weights
  • Machine-specific paths and workstation-only state

Repository Layout

  • models/: kept release checkpoint and index
  • third_party/Applio/: backend source tree without bundled runtime downloads
  • docs/assets/: project page media assets
  • data/source/: optional local dataset links only

Release Intent

This repository is designed as a practical training and evaluation scaffold for character singing voice conversion. The シリウスの心臓 example demonstrates one complete path through the framework, while the structure remains open for other songs, other guide vocals, and further model iterations.

Mayuri-Amadeus-Sing keeps the framework public and lightweight while leaving the dataset, model, and demo path concrete enough to reproduce.
